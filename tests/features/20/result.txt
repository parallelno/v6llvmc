// O16: Post-RA Store-to-Load Forwarding
// Primary case: multi_ptr_copy — high register pressure loop
void multi_ptr_copy(unsigned char *dst, unsigned char *src, unsigned char n) {
    unsigned char i;
    for (i = 0; i < n; i++) {
        dst[i] = src[i] + 1;
    }
}

        ex hl, de                       ; 1B   4cc

Note: c8080 uses static allocation (no stack frame), so has no spill/reload
overhead at all. The v6llvmc backend uses a stack-based calling convention,
which causes inherent overhead from spills. The O16 optimization reduces this.

== Summary ==
Post-RA Store-to-Load Forwarding (O16) eliminates 2 redundant RELOAD16 sequences
from the multi_ptr_copy loop body per iteration:

1. SPILL16 HL → RELOAD16 HL:  9-instruction sequence (11 bytes, 76 cycles)
   replaced by XCHG (1 byte, 4 cycles) — saves 10 bytes, 72 cycles

2. SPILL16 BC → RELOAD16 BC:  7-instruction sequence (9 bytes, 62 cycles)
   replaced by MOV E,C; MOV D,B (2 bytes, 8 cycles) — saves 7 bytes, 54 cycles

Total per-iteration savings: 12 instructions, 16 bytes, 116 cycles (18.8% fewer cycles)
