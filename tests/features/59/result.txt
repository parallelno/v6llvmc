Feature 59 — O77: V6C_STORE8_P Per-Shape Redesign
==================================================

Goal
----
Rewrite the priority-4 arm of `V6C_STORE8_P` (the i8 store-through-pointer
pseudo) to dispatch on `(AddrReg, A-liveness, dead-GR8 availability)`
instead of always emitting `PUSH PSW; MOV A,src; STAX; POP PSW`.

New shapes:
  - case 7  (addr=DE, A live, src!=A) — `XCHG; MOV M,partner(src); XCHG`
            via partner-MOV trick (B->B, C->C, H->D, L->E, D->H, E->L).
            3B / 16cc, fires unconditionally for any non-A src. Unlike
            the load (O76) the bypass is universally safe — body
            `MOV M,r` only reads a register, so XCHG2 always fully
            restores DE; no `src ∈ {D, E}` edge case.
  - case 6a (addr=BC, A live, SpareR avail) — `MOV spareR,A; MOV A,src;
            STAX B; MOV A,spareR`. 4B / 32cc.
  - case 6b (addr=BC, A live, no spare) — legacy `PUSH PSW` envelope.
            4B / 44cc (unchanged).


C test case (tests/features/59/v6llvmc.c)
-----------------------------------------
Six pinned-register DE-store probes + one BC-store probe + driver:

  void test_de_b(unsigned hl_keep, uint8_t *de_ptr, uint8_t a_keep,
                 uint8_t v_in) {
      register uint8_t v asm("B") = v_in;
      *de_ptr = v;
      asm volatile ("OUT 0xde" :: "a"(a_keep));
      asm volatile ("OUT 0xde" :: "r"(hl_keep));
  }
  // test_de_c : src pinned to C
  // test_de_h : src pinned to H, no hl_keep sink
  // test_de_l : src pinned to L, no hl_keep sink
  // test_de_d : src pinned to D — partner-MOV emits `MOV M, H`
  // test_de_e : src pinned to E — partner-MOV emits `MOV M, L`

  void test_bc_b(unsigned hl_keep, unsigned de_keep, uint8_t *bc_ptr,
                 uint8_t a_keep, uint8_t v_in) {
      register uint8_t v asm("B") = v_in;
      *bc_ptr = v;
      asm volatile ("OUT 0xde" :: "a"(a_keep));
      asm volatile ("OUT 0xde" :: "r"(hl_keep));
      asm volatile ("OUT 0xde" :: "r"(de_keep));
  }

  int main(int argc, char **argv) {
      uint8_t *p = (uint8_t *)g_buf;
      test_de_b(g_hl, p, g_a, g_v);
      test_de_c(g_hl, p, g_a, g_v);
      test_de_h(g_hl, p, g_a, g_v);
      test_de_l(g_hl, p, g_a, g_v);
      test_de_d(g_hl, p, g_a, g_v);
      test_de_e(g_hl, p, g_a, g_v);
      test_bc_b(g_hl, g_de, p, g_a, g_v);
      return 0;
  }

The c8080 reference (tests/features/59/c8080.c) is shape-equivalent
plain C — c8080 cannot pin physical registers, so it serves only as a
per-function cycle/byte anchor. Functions: store_use_b, store_use_c,
store_use_h, store_use_l, store_via_bc.

Note on v6llvmc free-list CC: for these signatures the i8 argument
`v_in` always arrives in B, so register pinning to B/C/H/L/D/E in
the source is a no-op at -O2 — the asm still uses B as src for every
DE test. Partner-MOV cases (src ∈ {H, L, D, E}) are exercised by the
lit MIR test in plan Step 3.3.


c8080 ASM — main + dependents (Z80 mnemonics → i8080 mnemonics)
---------------------------------------------------------------
main:
        LXI     H, g_buf
        SHLD    main_p
        LHLD    g_hl
        SHLD    __a_1_store_use_b
        LHLD    main_p
        SHLD    __a_2_store_use_b
        LDA     g_a
        STA     __a_3_store_use_b
        LDA     g_v
        CALL    store_use_b
        ; ... three more identical CC-spill blocks for c/h/l ...
        LHLD    g_hl
        SHLD    __a_1_store_via_bc
        LHLD    g_de
        SHLD    __a_2_store_via_bc
        LHLD    main_p
        SHLD    __a_3_store_via_bc
        LDA     g_a
        STA     __a_4_store_via_bc
        LDA     g_v
        CALL    store_via_bc
        LXI     H, 0
        RET

store_use_b:                            ; v already in A on entry
        STA     __a_4_store_use_b       ; spill v
        LHLD    __a_2_store_use_b       ; reload pointer into HL
        MOV     M, A                    ; the store
        LDA     __a_3_store_use_b       ; (dead — c8080 DSE quirk)
        LDA     __a_1_store_use_b       ; (u8)hl_keep
        STA     g_sink
        RET

store_use_c, store_use_h, store_use_l : same shape (h/l drop the
                                        hl_keep STA g_sink pair)
store_via_bc                          : same as store_use_b plus one
                                        extra LDA spill for de_keep


c8080 stats (per function, body + RET)
--------------------------------------
Timings (V6CInstructionTimings.md): LHLD/SHLD=3B/20cc, LDA/STA=3B/16cc,
LXI=3B/10cc, MOV M,r=1B/8cc, CALL=3B/24cc, RET=1B/12cc.

  store_use_b   ........... 17 B / 104 cc
  store_use_c   ........... 17 B / 104 cc
  store_use_h   ........... 14 B /  88 cc   (no hl_keep sink)
  store_use_l   ........... 14 B /  88 cc
  store_via_bc  ........... 20 B / 120 cc   (extra LDA for de_keep)
  main          ........... 121 B / 808 cc

The c8080 store proper (`*p = v`) costs `STA spill; LHLD ptr; MOV M, A`
= 7B / 44cc — c8080 must spill A and reload the pointer from a
static-stack slot per call. The store itself (`MOV M, A`) is 1B/8cc,
same as v6llvmc. Not directly comparable to v6llvmc's free-list
register CC where the pointer arrives in DE/BC and the value in B.
The relevant comparison is v6llvmc-old vs v6llvmc-new at the
store-expansion site (next two sections).

v6llvmc NEW ASM — store expansion (tests/features/59/v6llvmc_new01.asm)
-----------------------------------------------------------------------
test_de_b / test_de_c / test_de_d / test_de_e / test_de_h / test_de_l:

        XCHG
        MOV     M, B           ; partner(B)=B; for src=H emits MOV M,D, etc.
        XCHG
        ...OUT sinks...
        RET

test_bc_b:
        XCHG; SHLD .LLo61_0+1
        XCHG; SHLD .LLo61_1+1
        LXI     H, 2; DAD SP; MOV E, M
        MOV     D, A           ; SpareR = D (dead at the store site)
        MOV     A, E
        STAX    B
        MOV     A, D           ; restore A
        ...OUT sinks...
        RET

Correctness — XCHG bypass safety
--------------------------------
Unlike the load (O76), the store XCHG bypass is universally safe for
every non-A src. The body `MOV M, partner(src)` reads only a register
and writes memory — it never modifies DE/HL. The trailing XCHG
exactly inverts XCHG1, so DE and HL both end the sequence with their
pre-bypass values regardless of `src ∈ {D, E}` and regardless of
post-pseudo liveness. No RA-invariant argument required.
