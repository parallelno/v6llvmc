# O40: ADD16 DAD-Based Expansion — Result

## Status: PASS

## C test case

```c
volatile int sink_val;
__attribute__((noinline)) void use_val(int x) { sink_val = x; }
__attribute__((noinline)) int get_val(void)    { return sink_val; }

int nested_add(int n) {
    int a = get_val();
    int b = get_val();
    int c = a + b + n;
    use_val(c);
    return a + b;
}

int main(void) {
    int r = nested_add(5);
    return r;
}
```

## c8080 assembly (Z80 → i8080 mnemonics, function bodies only)

```asm
use_val: ; 28cc, 4B
    SHLD    sink_val                ; 20cc, 3B
    RET                             ; 8cc (Rcc? actually 12cc for RET)

get_val: ; 32cc, 4B
    LHLD    sink_val                ; 20cc, 3B
    RET                             ; 12cc, 1B

nested_add: ; 212cc, 27B
    SHLD    __a_1_nested_add        ; 20cc, 3B  (save n)
    CALL    get_val                 ; 24cc, 3B
    SHLD    nested_add_a            ; 20cc, 3B  (save a)
    CALL    get_val                 ; 24cc, 3B
    SHLD    nested_add_b            ; 20cc, 3B  (save b)
    LHLD    nested_add_a            ; 20cc, 3B
    XCHG                            ; 4cc, 1B
    LHLD    nested_add_b            ; 20cc, 3B
    DAD     DE                      ; 12cc, 1B  (HL = a + b)
    XCHG                            ; 4cc, 1B
    LHLD    __a_1_nested_add        ; 20cc, 3B
    DAD     DE                      ; 12cc, 1B  (HL = n + (a+b))
    SHLD    nested_add_c            ; 20cc, 3B  (save c)
    CALL    use_val                 ; 24cc, 3B  (live: HL=c)
    LHLD    nested_add_a            ; 20cc, 3B
    XCHG                            ; 4cc, 1B
    LHLD    nested_add_b            ; 20cc, 3B
    DAD     DE                      ; 12cc, 1B  (HL = a + b)
    RET                             ; 12cc, 1B

main: ; 60cc, 9B
    LXI     HL, 5                   ; 12cc, 3B
    CALL    nested_add              ; 24cc, 3B
    SHLD    main_r                  ; 20cc, 3B
    RET                             ; 12cc, 1B (note: also has init code)
```

## v6llvmc assembly (OLD — before O40, byte-chain expansion)

```asm
use_val: ; 32cc, 4B
    SHLD    sink_val                ; 20cc, 3B
    RET                             ; 12cc, 1B

get_val: ; 32cc, 4B
    LHLD    sink_val                ; 20cc, 3B
    RET                             ; 12cc, 1B

nested_add: ; 196cc, 22B
    MOV     D, H                    ; 8cc, 1B   (save n: DE = HL)
    MOV     E, L                    ; 8cc, 1B
    CALL    get_val                 ; 24cc, 3B  (a → HL)
    MOV     B, H                    ; 8cc, 1B   (save a: BC = HL)
    MOV     C, L                    ; 8cc, 1B
    CALL    get_val                 ; 24cc, 3B  (b → HL)
    MOV     A, L                    ; 8cc, 1B   ┐ byte chain: BC = HL + BC
    ADD     C                       ; 4cc, 1B   │ (a + b → BC)
    MOV     C, A                    ; 8cc, 1B   │
    MOV     A, H                    ; 8cc, 1B   │
    ADC     B                       ; 4cc, 1B   │
    MOV     B, A                    ; 8cc, 1B   ┘ 40cc, 6B
    MOV     A, C                    ; 8cc, 1B   ┐ byte chain: HL = BC + DE
    ADD     E                       ; 4cc, 1B   │ (a + b + n → HL)
    MOV     L, A                    ; 8cc, 1B   │
    MOV     A, B                    ; 8cc, 1B   │
    ADC     D                       ; 4cc, 1B   │
    MOV     H, A                    ; 8cc, 1B   ┘ 40cc, 6B
    CALL    use_val                 ; 24cc, 3B  (HL = c)
    MOV     H, B                    ; 8cc, 1B   (return a+b from BC)
    MOV     L, C                    ; 8cc, 1B
    RET                             ; 12cc, 1B
```

## v6llvmc assembly (NEW — with O40, DAD-based expansion)

```asm
use_val: ; 32cc, 4B
    SHLD    sink_val                ; 20cc, 3B
    RET                             ; 12cc, 1B

get_val: ; 32cc, 4B
    LHLD    sink_val                ; 20cc, 3B
    RET                             ; 12cc, 1B

nested_add: ; 152cc, 14B
    MOV     D, H                    ; 8cc, 1B   (save n: DE = HL)
    MOV     E, L                    ; 8cc, 1B
    CALL    get_val                 ; 24cc, 3B  (a → HL)
    MOV     B, H                    ; 8cc, 1B   (save a: BC = HL)
    MOV     C, L                    ; 8cc, 1B
    CALL    get_val                 ; 24cc, 3B  (b → HL)
    DAD     BC                      ; 12cc, 1B  ← Path B1-DE: HL = HL + BC (a+b)
    MOV     B, H                    ; 8cc, 1B   ← save result to BC
    MOV     C, L                    ; 8cc, 1B
    XCHG                            ; 4cc, 1B   ← Path B1-DE: XCHG (DE→HL)
    DAD     BC                      ; 12cc, 1B  ← DAD BC (HL = n + (a+b))
    CALL    use_val                 ; 24cc, 3B  (HL = c)
    MOV     H, B                    ; 8cc, 1B   (return a+b from BC)
    MOV     L, C                    ; 8cc, 1B
    RET                             ; 12cc, 1B
```

## Detailed improvement in nested_add

Two 16-bit additions were transformed:

### Addition 1: BC = HL + BC (a + b → BC)
| Version | Sequence                              | Cycles | Bytes |
|---------|---------------------------------------|--------|-------|
| OLD     | MOV A,L; ADD C; MOV C,A; MOV A,H; ADC B; MOV B,A | 40 | 6 |
| NEW     | DAD BC; MOV B,H; MOV C,L             |     28 |     3 |
| Saving  |                                       | **−12**| **−3**|

Path: B-general (DstReg=BC, one operand is HL → DAD BC; copy result to BC)

### Addition 2: HL = BC + DE ((a+b) + n → HL)
| Version | Sequence                              | Cycles | Bytes |
|---------|---------------------------------------|--------|-------|
| OLD     | MOV A,C; ADD E; MOV L,A; MOV A,B; ADC D; MOV H,A | 40 | 6 |
| NEW     | XCHG; DAD BC                          |     16 |     2 |
| Saving  |                                       | **−24**| **−4**|

Path: B1-DE (DstReg=HL, neither operand HL, one operand is DE →
XCHG brings DE into HL, DAD BC adds the other)

### main function also improved
| Version | Sequence (DE = HL + DE)               | Cycles | Bytes |
|---------|---------------------------------------|--------|-------|
| OLD     | MOV A,L; ADD E; MOV E,A; MOV A,H; ADC D; MOV D,A | 40 | 6 |
| NEW     | DAD DE; XCHG                          |     16 |     2 |
| Saving  |                                       | **−24**| **−4**|

Path: A1-DE (DstReg=DE, one operand is HL, HL dead after → DAD DE; XCHG)

## Comparison with c8080

| Metric          | c8080 nested_add | v6llvmc OLD | v6llvmc NEW |
|-----------------|------------------|-------------|-------------|
| Cycles          |              312 |         196 |         152 |
| Bytes           |               44 |          22 |          14 |

- v6llvmc keeps values live in registers (no SHLD/LHLD round-trips)
- c8080 recomputes a+b for return (two DAD+XCHG sequences vs register hold)
- O40 improves on already-superior register allocation with shorter ADD16 sequences
