v6llvmc.c (and equivalent c8080.c):

    extern unsigned short ext_sink16(unsigned short v);

    unsigned short sum_array(const unsigned short *a, unsigned char n) {
        unsigned short s = 0;
        for (unsigned char i = 0; i < n; i++)
            s += a[i] * 3 + 7;
        return s;
    }

    unsigned short poly(const unsigned short *a, unsigned char n) {
        unsigned short s = 0;
        for (unsigned char i = 0; i < n; i++) {
            unsigned short x = a[i];
            s += x * 5 + (x >> 1) + 3;
        }
        return s;
    }

    unsigned short g_arr[8];

    int main(int, char**) {
        g_arr[0..7] = 1..8;
        unsigned short r1 = sum_array(g_arr, 8);
        unsigned short r2 = poly(g_arr, 8);
        return ext_sink16(r1 + r2);
    }

What changed — and why
----------------------
Driver: O22 makes i16 ops cost 6× and i16 cmp cost 4× their i8 equivalents.
Two visible IR-level effects:

1. Loop counter narrowed i16 → i8.
   IndVarSimplify/LSR now sees that an i16 IV with i16 cmp costs 4
   per compare vs 1 for i8. It demotes the counter to i8, eliminating
   the 3-instruction `MOV A,B; ORA C; JNZ` zero-test in favour of
   `DCR A; JNZ`.

2. Lower spill pressure.
   Demoting the counter frees a register pair, so the per-iter spill
   set drops from 4×SHLD + PUSH/POP to 2×SHLD + 1×STA.

3. Pointer increment hoisted into reload (LHLD .LLo61_0+1 reloads
   the *post-incremented* HL; INX H executes on the reloaded value).
   No correctness cost — just less critical-path reshuffling.

Equivalent improvement applies to poly (file slimmed similarly).
