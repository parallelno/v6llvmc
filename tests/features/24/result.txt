# O41: Pre-RA INX/DCX Pseudo — Result

## Status: PASS

## C test case

```c
#include <stdint.h>
#define LEN 100

uint8_t array1[LEN];
uint8_t array2[LEN];

__attribute__((noinline))
void fill_array(uint8_t start_val) {
    for (uint8_t i = 0; i < LEN; ++i)
        array1[i] = start_val + i;
}

__attribute__((noinline))
void copy_loop(void) {
    for (uint8_t i = 0; i < LEN; ++i)
        array2[i] = array1[i];
}

__attribute__((noinline))
uint16_t add_small(uint16_t x) {
    return x + 2;
}

__attribute__((noinline))
uint16_t sub_small(uint16_t x) {
    return x - 1;
}

int main(void) {
    fill_array(10);
    copy_loop();
    uint16_t a = add_small(100);
    uint16_t b = sub_small(200);
    return array2[0] + array2[99] + (uint8_t)a + (uint8_t)b;
}
```

## Analysis

**What O41 fixes:**
Before O41, the LoopPointerInduction pass (O19) correctly recognized pointer
increments and produced `add i16 %ptr, 1` → DAD → INX via the post-RA peephole
(O29). However, the DAG combine unconditionally lowered `add i16 %ptr, 1` to
`V6CISD::DAD(ptr, LXI 1)`, which forced the register allocator to assign BC to
hold the constant `1`. The INX/DCX peephole then eliminated the DAD but left the
dead `LXI BC, 1` in the preheader, wasting 3 bytes + 12 cycles + tying up BC.

**How O41 works:**
The DAG combine now intercepts `ISD::ADD` / `ISD::SUB` with small constants
(±1..±3) **before** the DAD conversion. It emits `V6CISD::INX16` / `V6CISD::DCX16`
pseudo-instructions that carry the repeat count as a `TargetConstant` operand.
These pseudos expand post-RA to N copies of `INX rp` / `DCX rp`, with no register
allocation needed for the constant. The `$dst = $src` constraint ensures tied
register assignment. The pseudos declare no `Defs` (no A/FLAGS clobber).

**Key improvements:**
1. **Code size:** −6B total (two dead `LXI BC, 1` eliminated)
2. **Register pressure:** BC freed in loops — available for other live ranges
3. **Cycles:** −24cc (marginal, from skipping dead LXI setup)
4. **Indirect benefit:** Freeing BC enables better RA choices in complex loops
   (verified: spill-forwarding.ll test changed register assignment beneficially)

**Implementation note:**
The DAG combine must use `DAG.getTargetConstant()` (not `DAG.getConstant()`)
for the count operand, because the TableGen patterns use `timm` which matches
`ISD::TargetConstant` nodes. Using `getConstant()` produces "Cannot select"
ISel failures.
