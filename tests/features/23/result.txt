# O20: Honest Store/Load Pseudo Defs — Result

## Status: PASS

## C test case

```c
#include <stdint.h>
#define LEN 100

uint8_t array1[LEN];
uint8_t array2[LEN];

__attribute__((noinline))
void fill_array(uint8_t start_val) {
    for (uint8_t i = 0; i < LEN; ++i)
        array1[i] = start_val + i;
}

__attribute__((noinline))
void copy_array(void) {
    for (uint8_t i = 0; i < LEN; ++i)
        array2[i] = array1[i];
}

int main(void) {
    fill_array(42);
    copy_array();
    return array2[0] + array2[99];
}
```

## c8080 assembly (Z80 → i8080 mnemonics, function bodies only)

```asm
fill_array:                             ; 34B
    STA     __a_1_fill_array            ; 16cc, 3B  save arg
    XRA     A                           ; 4cc, 1B
    STA     fill_array_i                ; 16cc, 3B  i = 0
l_0:
    CPI     100                         ; 8cc, 2B
    RNC                                 ; 8cc/16cc, 1B  exit if i >= 100
    LXI     HL, __a_1_fill_array        ; 12cc, 3B
    ADD     M                           ; 8cc, 1B   A = i + start_val
    LXI     DE, array1                  ; 12cc, 3B
    LHLD    fill_array_i                ; 20cc, 3B  HL = i (zero-extended)
    MVI     H, 0                        ; 8cc, 2B
    DAD     DE                          ; 12cc, 1B  HL = array1 + i
    MOV     M, A                        ; 8cc, 1B   store
    LDA     fill_array_i                ; 16cc, 3B
    INR     A                           ; 8cc, 1B
    STA     fill_array_i                ; 16cc, 3B  i++
    JMP     l_0                         ; 12cc, 3B
    ; Loop body: 148cc, 27B

copy_array:                             ; 37B
    XRA     A                           ; 4cc, 1B
    STA     copy_array_i                ; 16cc, 3B
l_3:
    CPI     100                         ; 8cc, 2B
    RNC                                 ; 8cc/16cc, 1B
    LXI     DE, array1                  ; 12cc, 3B
    LHLD    copy_array_i                ; 20cc, 3B
    MVI     H, 0                        ; 8cc, 2B
    DAD     DE                          ; 12cc, 1B
    MOV     A, M                        ; 8cc, 1B   load src
    LXI     DE, array2                  ; 12cc, 3B
    LHLD    copy_array_i                ; 20cc, 3B
    MVI     H, 0                        ; 8cc, 2B
    DAD     DE                          ; 12cc, 1B
    MOV     M, A                        ; 8cc, 1B   store dst
    LDA     copy_array_i                ; 16cc, 3B
    INR     A                           ; 8cc, 1B
    STA     copy_array_i                ; 16cc, 3B
    JMP     l_3                         ; 12cc, 3B
    ; Loop body: 188cc, 33B

main:                                   ; 21B
    MVI     A, 42                       ; 8cc, 2B
    CALL    fill_array                  ; 24cc, 3B
    CALL    copy_array                  ; 24cc, 3B
    LHLD    array2                      ; 20cc, 3B
    MVI     H, 0                        ; 8cc, 2B
    XCHG                                ; 4cc, 1B
    LHLD    array2+99                   ; 20cc, 3B
    MVI     H, 0                        ; 8cc, 2B
    DAD     DE                          ; 12cc, 1B
    RET                                 ; 12cc, 1B
```

## Detailed improvement in fill_array

The main O20 benefit: removing BC→HL copy from every iteration.

### Before O20 (loop body = 64cc):
```
MOV H, B    ; 8cc ┐ wasted: copy pointer BC→HL
MOV L, C    ; 8cc ┘ because RA declares HL clobbered by pseudo
MOV M, E    ; 8cc   store via HL
INX BC      ; 8cc   pointer++
```
RA kept pointer in BC because it believed HL was clobbered by V6C_STORE8_P.

### After O20 (loop body = 56cc):
```
MOV A, L    ; 8cc   load value from L into A
STAX DE     ; 8cc   store directly via STAX (addr=DE, val=A)
INX DE      ; 8cc   pointer++
```
With Defs=[HL] removed, RA is free to keep value in L (sub-register of HL).
The pseudo expands to STAX DE (priority 3 path: addr=DE, A dead after store).

### Savings per iteration: 64cc → 56cc = -8cc
### Total savings for 100 iterations: -800cc + 12cc setup = -788cc (12.2%)

## copy_array analysis

copy_array changed register assignments (BC,DE → HL,DE) but the loop body
is the same 56cc. The +12cc overhead is from a dead `LXI BC, 1` instruction
in the preheader — an artifact of the cross-BB INX peephole optimization
that doesn't erase constants from predecessor blocks. This is negligible
(one-time setup cost, not per-iteration).
