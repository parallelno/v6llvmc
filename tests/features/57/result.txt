Feature 57 — O75: Flag-Producing Arithmetic SDNodes
====================================================

Goal
----
Always lower i8 ADD/SUB/AND/OR/XOR through new V6CISD::*F nodes that
expose FLAGS as a second SSA result, so a downstream BR_CC / SELECT_CC
that compares the result to zero (EQ/NE) can consume those FLAGS
directly instead of emitting a redundant CPI 0 / ORA A.

Phases
------
- Phase A: rewire FLAGS as an SSA i8 value (replace MVT::Glue chain).
- Phase B: 12 V6CISD::*F nodes (ADDF/SUBF/ANDF/ORF/XORF, *_IMM, INCF,
           DECF) + setOperationAction(...,Custom) on i8 ADD/SUB/AND/OR/XOR
           + EQ/NE-against-zero short-circuit in LowerBR_CC / LowerSELECT_CC.

C test case (tests/features/57/v6llvmc.c)
-----------------------------------------
extern volatile u8 g_sink;
u16 dec_loop(u8 n) {
    u16 sum = 0;
    while (n) { g_sink = n; sum += n; --n; }
    return sum;
}
u8  mask_test(u8 x)        { return (x & 0x0F) == 0 ? 1 : 0; }
u8  xor_test (u8 x, u8 y)  { u8 z = x ^ y; g_sink = z; return z != 0; }
u8  sub_test (u8 x)        { u8 z = x - 5; g_sink = z; return z != 0; }
u16 dec_loop_used(u8 n)    { /* same loop, returns sum + (u16)n */ }

volatile u8  g_n  = 7, g_x = 0x33, g_y = 0x33, g_outb;
volatile u16 g_out;

int main(int, char **) {
    g_out  = dec_loop(g_n);
    g_outb = mask_test(g_x);
    g_outb = xor_test(g_x, g_y);
    g_outb = sub_test(g_x);
    g_out  = dec_loop_used(g_n);
    return 0;
}

Notes
-----
- v6llvmc's `dec_loop` is much smaller than c8080's because the loop
  body is fully register-resident; c8080 spills `n` and `sum` to a
  static stack across each iteration.
- `dec_loop` is unchanged by O75 because the C source forces `n` into
  A every iteration (via `g_sink = n;`). `DCR A` already produces
  FLAGS that the loop-back JNZ consumes — no extra `CPI 0` was being
  emitted to begin with. The Acc-pin here is structural, not a CPI 0
  issue.
- `xor_test` is the headline win: removing the FLAGS glue (Phase A)
  freed the scheduler to hoist `STA g_sink` past the conditional,
  which let the *F fold (Phase B) consume `XRA L`'s FLAGS without
  the previous `MOV L,A` / `MOV A,L` spill round-trip.
- `mask_test` and `sub_test` get +1 byte each: they no longer rely
  on the implicit `A = 0` post-condition of a taken JZ (so `INR A`
  / `MOV L,A` become `MVI A,1` / `MVI L,0`). This is offset by the
  inline savings in `main`.
- Net win across .text body: -20 cycles / -1 byte.

Implementation summary
----------------------
- 12 new V6CISD nodes (each producing (i8 value, i8 flags)):
    ADDF, SUBF, ANDF, ORF, XORF,
    ADDF_IMM, SUBF_IMM, ANDF_IMM, ORF_IMM, XORF_IMM,
    INCF, DECF

- TableGen patterns rewritten to multi-result form:
    ADDr / SUBr / ANAr / ORAr / XRAr,
    ADI  / SUI  / ANI  / ORI  / XRI,
    INRr / DCRr,
    V6C_ADD_M_P / V6C_SUB_M_P / V6C_ANA_M_P / V6C_ORA_M_P / V6C_XRA_M_P,
    V6C_INR_M_P / V6C_DCR_M_P
  All emit `[(set i8:$dst, FLAGS, (V6C*f ...))]`.

- LowerArithF: setOperationAction(Custom) on i8 ADD/SUB/AND/OR/XOR.
  Always rewrites to *F variants. Special case: `XOR x, -1` stays as
  `ISD::XOR` so the existing CMA pattern matches (CMA does NOT
  affect FLAGS, so it intentionally stays out of the *F family).

- LowerBR_CC / LowerSELECT_CC: when LHS.getOpcode() is in
  [V6CISD::ADDF .. V6CISD::DECF] and the comparison is EQ/NE against
  zero, consume LHS.getValue(1) directly as FLAGS — no redundant
  CMP / CPI 0.
