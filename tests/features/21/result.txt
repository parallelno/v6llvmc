# O10: Static Stack Allocation for Non-Reentrant Functions — Result

## Status: PASS

## C test case

```c
volatile int sink_val;
__attribute__((noinline)) void use_val(int x) { sink_val = x; }
__attribute__((noinline)) int get_val(void)    { return sink_val; }

int heavy_spill(int a, int b) {
    int x = a + 1;
    int y = b + 2;
    int z = a + b;
    use_val(x);
    use_val(y);
    use_val(z);
    return x + y + z;
}

int nested_calls(int n) {
    int a = get_val();
    int b = get_val();
    int c = a + b + n;
    use_val(c);
    return a + b;
}

int main(void) {
    int r;
    r = heavy_spill(10, 20);
    use_val(r);
    r = nested_calls(5);
    use_val(r);
    return r;
}
```

## c8080 assembly (Z80 → i8080 mnemonics, function bodies only)

```asm
use_val:
    SHLD    __a_1_use_val
    SHLD    sink_val
    RET

get_val:
    LHLD    sink_val
    RET

heavy_spill:
    SHLD    __a_2_heavy_spill
    LHLD    __a_1_heavy_spill
    INX     HL
    SHLD    heavy_spill_x
    LHLD    __a_2_heavy_spill
    INX     HL
    INX     HL
    SHLD    heavy_spill_y
    LHLD    __a_1_heavy_spill
    XCHG
    LHLD    __a_2_heavy_spill
    DAD     DE
    SHLD    heavy_spill_z
    LHLD    heavy_spill_x
    CALL    use_val
    LHLD    heavy_spill_y
    CALL    use_val
    LHLD    heavy_spill_z
    CALL    use_val
    LHLD    heavy_spill_x
    XCHG
    LHLD    heavy_spill_y
    DAD     DE
    XCHG
    LHLD    heavy_spill_z
    DAD     DE
    RET

nested_calls:
    SHLD    __a_1_nested_calls
    CALL    get_val
    SHLD    nested_calls_a
    CALL    get_val
    SHLD    nested_calls_b
    LHLD    nested_calls_a
    XCHG
    LHLD    nested_calls_b
    DAD     DE
    XCHG
    LHLD    __a_1_nested_calls
    DAD     DE
    SHLD    nested_calls_c
    CALL    use_val
    LHLD    nested_calls_a
    XCHG
    LHLD    nested_calls_b
    DAD     DE
    RET

main:
    LXI     HL, 0
    SHLD    __a_1_main
    SHLD    __a_2_main
    LXI     HL, 10
    SHLD    __a_1_heavy_spill
    LXI     HL, 20
    CALL    heavy_spill
    SHLD    main_r
    CALL    use_val
    LXI     HL, 5
    CALL    nested_calls
    SHLD    main_r
    CALL    use_val
    LHLD    main_r
    RET
```

## Comparison

### heavy_spill (the spill-heavy function where O10 applies)

| Metric              | c8080 (static) | v6llvmc baseline (SP) | v6llvmc O10 (static) |
|---------------------|----------------|-----------------------|----------------------|
| Cycles              |            436 |                   768 |                  364 |
| Bytes               |             62 |                   102 |                   49 |
| DAD SP instructions |              0 |                     7 |                    0 |
| Prologue            |           none |      5 instr (64cc)   |                 none |
| Epilogue            |           none |      6 instr (52cc)   |                 none |

### Key improvements in heavy_spill with O10:
- vs baseline: **-404cc (−53%)**, **-53B (−52%)**
- vs c8080:    **-72cc (−17%)**, **-13B (−21%)**

### Whole-program totals:

| Compiler         | Cycles | Bytes |
|------------------|--------|-------|
| c8080 (static)   |   1128 |   159 |
| v6llvmc baseline |   1354 |   187 |
| v6llvmc + O10    |    950 |   134 |

- O10 vs baseline: **-404cc (−30%)**, **-53B (−28%)**
- O10 vs c8080:    **-178cc (−16%)**, **-25B (−16%)**

## Notes

- Only heavy_spill benefits from O10 (it's the only function with spill slots).
- nested_calls and main have no spills — register allocation alone is sufficient.
- main inlines heavy_spill and nested_calls at -O2, constant-folding the results.
- c8080 uses a shared static stack for all functions (all variables in BSS, 18B).
- v6llvmc O10 allocates per-function: __v6c_ss.heavy_spill (4B in BSS).
