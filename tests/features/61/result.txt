Feature 61 — O79: MVI R,NN + ALU R → ALU-immediate fold
=======================================================

Summary
-------
Post-RA peephole `foldMviAluImm` rewrites the two-instruction sequence
`MVI R, NN` followed by `ALU R` (where R is a non-A GR8) into the
single-instruction ALU-immediate `<ALUI> NN` on the accumulator. The
fold is gated on R being dead after the consumer and forward-scans
across only side-effect-free, non-clobbering instructions. The pass
preserves any pre-instruction symbol attached to the original MVI,
which means O61 patched-reload landing pad labels (`.LLo61_<n>:`) and
their backing relocations (`STA <Sym>+1` / `SHLD <Sym>+1`) survive
the fold.

The interesting trigger in this test is `fold_spill`: with eight i8
parameters, RA has to spill at least one carrier through a non-A
GR8, which O61 lowers to a patched landing pad whose reload site is
`MVI L, 0; XRA L`. That sequence is exactly the new fold target.

The simple `fold_*` functions are already lowered as ALU-immediate by
ISel, so O79 does not change them — they are listed for completeness
to document that the fold does not regress the trivial cases.


================================================================
1. C test case (tests/features/61/v6llvmc.c)
================================================================

  typedef unsigned char u8;
  extern volatile u8 g_sink;

  u8 fold_add(u8 a) { return a + 0x0F; }
  u8 fold_sub(u8 a) { return a - 0x05; }
  u8 fold_and(u8 a) { return a & 0xF0; }
  u8 fold_or (u8 a) { return a | 0x01; }
  u8 fold_xor(u8 a) { return a ^ 0x55; }
  int fold_cmp(u8 a) { return (a == 0x42) ? 1 : 0; }
  u8 fold_chain(u8 a) { return ((a + 5) & 0xF0) ^ 0x10; }

  u8 fold_spill(u8 x0, u8 x1, u8 x2, u8 x3,
                u8 x4, u8 x5, u8 x6, u8 x7) {
      u8 s = x0;
      s ^= x1; s ^= x2; s ^= x3;
      s ^= x4; s ^= x5; s ^= x6;
      s ^= x7;
      return s;
  }

  volatile u8 g_sink;

  int main(int argc, char **argv) {
      (void)argc; (void)argv;
      g_sink = fold_add(0x10);
      g_sink = fold_sub(0x10);
      g_sink = fold_and(0xAB);
      g_sink = fold_or(0xAA);
      g_sink = fold_xor(0xAA);
      g_sink = (u8)fold_cmp(0x42);
      g_sink = fold_chain(0x07);
      g_sink = fold_spill(1, 2, 3, 4, 5, 6, 7, 8);
      return 0;
  }

fold_add:                ; +F
        STA  __a_1_fold_add
        ADI  0x0F
        RET

        STA  __a_1_fold_sub

        RET

fold_and:                ; &F0
        STA  __a_1_fold_and
        ANI  0xF0
        RET

fold_or:                 ; |1
        STA  __a_1_fold_or
        ORI  0x01
        RET

fold_xor:                ; ^55
        STA  __a_1_fold_xor
        XRI  0x55
        RET

fold_cmp:                ; == 0x42 ? 1 : 0  (returns in HL)
        STA  __a_1_fold_cmp
        CPI  0x42
        JNZ  l_0
        LXI  H, 1
        RET
l_0:    LXI  H, 0
        RET

fold_chain:              ; ((a+5)&F0)^10
        STA  __a_1_fold_chain
        ADI  5
        ANI  0xF0
        XRI  0x10
        RET

fold_spill:              ; static-stack accumulator across __a_*
        STA  __a_8_fold_spill
        LDA  __a_1_fold_spill
        STA  fold_spill_s
        LXI  H, fold_spill_s
        LDA  __a_2_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_3_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_4_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_5_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_6_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_7_fold_spill
        XRA  M
        MOV  M, A
        LDA  __a_8_fold_spill
        XRA  M
        MOV  M, A
        RET

Net change for this feature:
  -1 byte and -4 cycles per dynamic execution of `fold_spill`.

The four-cycle / one-byte saving is per O61 patched-reload landing
pad whose reload feeds an ALU-r consumer; the win scales with the
number of such non-A i8 spills the program makes in inner loops.
