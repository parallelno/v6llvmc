O17 Redundant Flag-Setting Elimination — Result

== C Test Case ==

volatile unsigned char g_port;

void countdown(unsigned char n) {
    while (n != 0) {
        g_port = n;
        n--;
    }
}

void countup(unsigned char start) {
    unsigned char i = start;
    while (i != 0) {
        g_port = i;
        i++;
    }
}

int xor_test(int a, int b) {
    int r = a ^ b;
    if (r == 0)
        return 1;
    return 0;
}

int sub_test(int a, int b) {
    int r = a - b;
    if (r == 0)
        return 1;
    return 0;
}

int main(void) {
    countdown(5);
    countup(250);
    xor_test(5, 5);
    sub_test(10, 10);
    return 0;
}

== Improvement ==

  Savings per instance: 4cc + 1B (redundant ORA A eliminated)
  Pattern: DCR A; ORA A; JNZ → DCR A; JNZ
  Applies to any ALU-writes-A instruction followed by ORA A / ANA A.

  v6llvmc countdown loop vs c8080:
    v6llvmc: 36cc/iter (after O17)
    c8080:   80cc/iter (static stack stores on every iteration)
    v6llvmc is 2.2× faster per iteration

  Note: O17 generalizes beyond the DCR+ORA A pattern handled by O18's
  foldCounterBranch. Any ALU instruction writing A (ADD, SUB, ANA, ORA,
  XRA, ADI, SUI, INR A, DCR A, etc.) followed by ORA A / ANA A is caught.
